• NEW
    zsim: The customMemory.slots[].banks[] configuration has been updated. You can now define custom memory models that emulate bank-switched memory and assemble code using sjasmplus with your own memory layout. Breakpoints and source-level debugging are supported not only for predefined bank scenarios, but also for custom memory bank setups. See this example project and the migration guide for details.
  • NEW
    LOGPOINT: New variables to log T-states, cpu frequency and memory slots (banks).
  • CHANGED
    The output logs for "DeZog zsim: Custom Code" and "DeZog zsim: Hardware" have been consolidated into a single log named "DeZog zsim". This change simplifies viewing related logs together.
